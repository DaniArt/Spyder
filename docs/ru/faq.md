# FAQ

## Не находит элемент при наведении

Причины:
- UI может быть отрисован как единый surface и не раскрывать детей через UIA/MSAA.
- Win32 hit-test видит только контейнерный HWND.
- Элемент может быть временным (создается/уничтожается на hover).

Рекомендации:
- Делай capture (hotkey/drag), а не только hover.
- Для VCL включи VCL интроспекцию.
- Включи логирование и смотри, какая команда падает.

## У элемента нет HWND

Non-HWND контролы рисуются родителем и не имеют отдельного Win32 handle. Для них:
- Win32/UIA может показывать только контейнер,
- нужен toolkit-specific путь (VCL интроспекция).

## VCL цепочка короткая

Типовые причины:
- UI поток цели не прокачивает сообщения (dispatch не выполняется).
- Дерево контролов меняется во время обхода.
- Сработали safety-лимиты по глубине/узлам.

Попробуй:
- повторить capture на “спокойном” UI,
- уменьшить активность окна на время захвата,
- использовать лог для анализа.

## Где логи

- `%LOCALAPPDATA%\Spyder\logs\spyder.log`
- Нативные dev-логи могут лежать в `VclHook/logs/`.

## Publish падает с “Access denied”

Обычно `Spyder.exe` еще запущен. Закрой процесс, очисть publish и повтори (см. `docs/ru/installation.md`).
